🤖 O que é o Otimize Qualidade?
Ferramenta de automação de testes funcionais CRUD para o sistema SEI.
Abre o Google Chrome automaticamente, faz login no SEI, navega até a tela escolhida e executa Inserir → Alterar → Excluir como um usuário real — verificando se cada operação foi concluída com sucesso.
🔍 Escanear Tela
Você aponta a URL de uma tela do SEI. O bot abre o Chrome, analisa todos os campos, labels e botões e gera automaticamente um arquivo de cenário JSON pronto para usar. Não precisa programar nada.
▶ Executar Cenário
O bot lê o cenário JSON, abre o Chrome, faz login e executa inserir → alterar → excluir → campos obrigatórios. Cada etapa é registrada com resultado (passou/falhou), duração e screenshot em caso de erro.
📋 Relatórios
Ao final de cada execução são gerados dois arquivos: um JSON estruturado e um HTML visual com tabela de etapas, badges coloridos por operação e screenshots expansíveis.
🧠 SmartLocator
O bot usa 7 estratégias em cascata para encontrar campos na tela (CSS, XPath, label, placeholder, aria-label, texto do botão, classe CSS). Se a tela mudar, ele se adapta automaticamente.
🚦 Fluxo de uso passo a passo
Primeira vez com uma nova tela
1
Aba Escanear — preencha a URL do SEI, usuário, senha e a URL relativa da tela (ex: visaoAdministrativo/academico/cursoCons.xhtml). Dê um ID e descrição ao cenário.
2
Clique "Escanear Tela" — o Chrome abre, o bot faz login, navega para a tela, clica em "Novo", detecta todos os campos e gera o JSON. O Chrome fecha automaticamente.
3
Preencha os Dados de Teste — aparece uma tabela com todos os campos detectados. Coluna Valor (Inserir) = o que vai ser cadastrado. Coluna Novo Valor (Alterar) = o que vai ser alterado.
4
Escolha o Campo Chave — campo único pelo qual o bot vai localizar o registro para alterar/excluir. Ex: "Nome" ou "Código".
5
Salvar e Executar — o cenário é salvo e o teste começa. Acompanhe o Log de Execução no painel direito em tempo real.

Um campo é confirmado como obrigatório somente se o sistema bloquear o salvamento e exibir um indicador de erro naquele campo específico quando ele está vazio. Se o sistema salvar sem o campo preenchido, o campo é reportado como NÃO confirmado — indicando uma falha na validação do sistema. Esta operação é executada após Excluir, utilizando um novo registro de teste.
O bot usa o valor {{timestamp}} nos campos de texto para garantir um nome único a cada execução (ex: "Curso Teste Bot 12345"). Isso evita conflitos com registros já existentes e garante que o alterar/excluir encontrem exatamente o registro criado naquela execução.
📄 Estrutura do Cenário JSON
Cada cenário é um arquivo .json salvo em cenarios/. Pode ser editado manualmente a qualquer momento.
CampoDescrição
idIdentificador único do cenário, sem espaços. Nome do arquivo JSON.
descricaoNome legível exibido na lista e nos relatórios.
moduloGrupo para organizar cenários (Acadêmico, RH, Financeiro…).
telaURL relativa da tela de listagem (Cons). Ponto de partida de todas as operações.
tela_formURL do formulário quando "Novo" redireciona para outra página. Preenchido automaticamente pelo scanner.
campo_chaveCampo usado para buscar o registro na tela de listagem durante alterar/excluir.
operacoes_disponiveisLista das operações que o cenário suporta: inserir, alterar, excluir.
inserir / alterar / excluirObjeto com lista de acoes (o que fazer) e validacoes (como verificar sucesso).
dados_testeMapa de valores customizados. Preenchido via interface ao escanear.

🏗️ Arquitetura e Tecnologias
🐍 servidor.py — Python
Servidor HTTP leve (http.server) na porta 9871. Recebe requisições do frontend, lança o bot Java como subprocess e transmite o log em tempo real via SSE (Server-Sent Events). Compila o bot com Maven automaticamente se o JAR não existir.
☕ bot-qualidade — Java 21
Fat JAR gerado pelo Maven. Dois modos: -m scan (escaneia tela) e -m executar (roda cenário). Usa Selenium 4.20 + WebDriverManager (baixa o ChromeDriver automaticamente). JSON via Jackson 2.17.
🗂️ Arquivos gerados
cenarios/{id}.json — cenário de teste
relatorios/relatorio-{id}.json — resultado estruturado
relatorios/relatorio-{id}.html — dashboard visual
locators.json — seletores customizados
🔍 SmartLocator — 7 estratégias
1. Seletores CSS/XPath do cenário
2. Texto do <label> (case-insensitive)
3. Atributo placeholder
4. Atributo name / id parcial
5. Atributo aria-label
6. Texto do botão (com aliases)
7. Classe CSS do botão (btn-gravar, btn-excluir…)

💡 Dicas e solução de problemas
SituaçãoO que fazer
Bot não iniciaClique 🔄 Recompilar na topbar. O botão Bot ficará verde após compilação bem-sucedida.
Login falhouVerifique URL, usuário e senha. A URL deve terminar com /. Teste abrindo o SEI manualmente no Chrome.
Campo não encontradoO log mostrará os labels e botões visíveis na tela. Use essas informações para ajustar o campo label no JSON do cenário, ou re-escanear a tela.
Alterar/Excluir não acha o registroVerifique se o Campo Chave está correto. O valor digitado na busca deve aparecer na tela de listagem. Use o botão ✏️ Editar Dados para ajustar.
Tela com abas (rich:tabPanel)O scanner detecta abas automaticamente. Se falhar, re-escanear costuma resolver.
Formulário não abre ao clicar NovoAlguns SEIs usam JavaScript para abrir o form. O bot tenta 3 estratégias (click(), navegação direta, jsClick). Se falhar, informe a URL do formulário manualmente em tela_form no JSON.
Cenário com muitas falhasExecute só o Inserir primeiro. Se passar, adicione Alterar. Isso facilita identificar em qual operação está o problema.
